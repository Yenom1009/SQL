## 领导方面：

前置的认为所有我们指定了几个user作为公司人事系统负责人或公司高管（anyway）,反正就是能在系统里修改增删的人。

- [x] 领导可以**查看所有的员工信息**

- [x] 领导可以**修改员工岗位和薪资**（这里要注意在“岗位-薪资变动表”里变动的记录要能够在员工信息表里也改动员工岗位和薪资）

  而且要有检查机制，每次修改后岗位和薪资要满足min和max匹配

  如果修改了部门负责人的岗位，不允许修改，必须先在部门表中修改负责人信息才能修改原负责人岗位

- [x] 领导可以**写请假申请**（这里有点难以理解，就是说员工先和自己的上司申请请假通过后，有上司来录入该员工的请假信息，否则员工可以自己乱填就不好了）

- [x] 领导可以**新增和删除员工**（如果删除员工，要先把岗位工资改成“空”，然后删掉，同时员工的请假、考勤、变岗记录都被删除、用户也被删除）

- [x] 领导可以**新增岗位**

- [x] 领导可以**新增部门**，**修改部门负责人**



## 登陆页面：
